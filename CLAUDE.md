# cc-slides — AI セールススライド自動生成システム

サービス仕様書（Markdown）を渡すだけで、プロ品質のセールススライド画像を自動生成するツール。

---

## Claudeへの指示

ユーザーがこのプロジェクトを開いたら、以下のステップを順番に能動的に案内する。
ユーザーが何も言わなくても、まずStep 1から始める。

---

## Step 1：セットアップ確認

以下を順番にチェックし、問題があればその場で解決してから次に進む。

> 以下のコマンドはWindows（Git Bash / PowerShell）・Mac/Linuxどちらでも同じ書き方で動作します。
> シェル差分が出る場合は併記しています。

### 1-A. node_modules の確認

```bash
ls slides/engine/node_modules
```

存在しなければ実行：
```bash
cd slides/engine && npm install
```

### 1-B. .env の確認

`.env` ファイルが存在するか確認：

```bash
# Mac / Linux / Git Bash
ls -la | grep .env

# Windows PowerShell
Get-ChildItem -Force | Select-String "\.env"
```

なければ作成を案内：

```bash
# Mac / Linux / Git Bash
cp .env.example .env

# Windows PowerShell
Copy-Item .env.example .env
```

`.env` を開いて以下の2つのキーが入力済みか確認する：
- `GOOGLE_AI_API_KEY` — 取得先: https://aistudio.google.com/apikey
- `ANTHROPIC_API_KEY` — 取得先: https://console.anthropic.com/

キーが空欄なら、ユーザーにキーの入力を促す。詳細手順は `setup/gemini-api.md` を参照。

セットアップが完了したら、以下の全体の流れを案内してから Step 2 へ進む：

```
準備完了です。以下の流れで進めます。

① 生成パラメータを3つ決める（訴求軸・ターゲット・スタイル）
② スライド構成を確認・調整する
③ テキスト設計案を生成・内容を確認する（Claude API・1〜2分）
④ スライド画像を生成する（Gemini API・5〜10分）

では①から始めます。
```

## Step 2：生成パラメータを決める

以下の4項目を**1つずつ順番に**聞く。複数まとめて聞かない。

**① 訴求軸**
```
どの訴求軸でスライドを作りますか？

1. 収入アップ（「稼ぎたい」層向け）
2. 時間自由（「会社に縛われたくない」層向け）
3. スキル習得（「AIを使いこなしたい」層向け）
4. 副業スタート（「何か始めたい」層向け）
5. AI人材になる（「市場価値を上げたい」層向け）
6. その他（自由入力）
```

**② ターゲット属性**
```
誰に向けて訴えますか？

1. 副業未経験の会社員（デフォルト）
2. フリーランス
3. 経営者・起業家
4. 主婦・主夫
5. その他（自由入力）
```

**③ スタイル**
```
デザインスタイルを選んでください。

manus     … 薄ブルー×ネイビー、洗練されたビジネス感
yellow    … イエロー×チャコール、鋭角デザインのビジネス感
teal      … ティール×オレンジ、データ重視のモダンビジネス感
premium   … ディープネイビー×ゴールド、高級セミナー感
corporate … 白×ティール×パープル、日本企業の事業説明資料風

迷ったら「おまかせ」でOKです。
```

**④ デモコンテンツ**
```
スライド8・9はデモ動画の導入ページです。
何のデモを見せますか？2つ教えてください。

例:
- スライド自動生成システム
- 案件探索自動化ツール

自由に入力してください（ツール名・機能名・システム名など）。
```

4項目が揃ったら `input/brief-template.md` の「## 生成パラメータ」欄と「## デモコンテンツ」欄を更新する。

---

```

削除・追加・変更があれば教えてください。問題なければ「はい」でテキスト設計案の生成に進みます。

08. デモ①（スライド自動生成）
09. デモ②（案件探索自動化）
10. 講師①：みかみ 実績
11. 講師②：佐藤将司 実績
12. 1期生の実績
13. 橋渡し（あなたにもできる）
14. 競合との差別化
15. 受講フロー（4ステップ）
16. コンテンツ①：講義13テーマ
17. コンテンツ②：ブートキャンプ
18. コンテンツ③：コミュニティ
19. 本体合計（¥649,000相当）
20. 特典①：社内システム12種
21. 特典②③：1on1・補講
22. 総合計（¥2,485,000相当）
23. 価格提示（¥2,485,000 → 198,000円）

```

「はい」または修正が出たら対応し、確定枚数を `input/brief-template.md` の「### 画像枚数」に書き込んで Step 3 へ進む。

## Step 3：テキスト設計案の生成（plan.js）

**確認なしに勝手に実行してはいけない。**

「はい」をもらった時点で実行する：

```bash
node slides/engine/plan.js --brief input/brief-template.md
```

完了したら `slides/image-design.json` を読み取り、全スライドの内容を以下の形式で表示する：

```
テキスト設計案が完成しました。
画像生成の前に内容をご確認ください。

━━━━━━━━━━━━━━━━━━━━━━━
Slide 01 ／ <役割・セクション名>
━━━━━━━━━━━━━━━━━━━━━━━
メイン：<main テキスト>
サブ　：<sub テキスト>
要素　：<other テキストを「／」区切りで列挙>

━━━━━━━━━━━━━━━━━━━━━━━
Slide 02 ／ <役割・セクション名>
━━━━━━━━━━━━━━━━━━━━━━━
メイン：<main テキスト>
サブ　：<sub テキスト>
要素　：<other テキストを「／」区切りで列挙>

…（全枚数分）

修正したいスライドがあれば番号と内容を教えてください。
なければ「はい」でスライド画像の生成（Step 4）に進みます。
```

修正がある場合は `slides/image-design.json` の該当スライドを編集してから進む。

---

## Step 4：スライド画像の生成（generate.js）

**「はい」を得てから実行する。確認なしに勝手に実行してはいけない。**

```bash
node slides/engine/generate.js --design slides/image-design.json --out slides/output
```

実行中の流れ（ユーザーに説明する）：
1. Gemini API が各スライドを画像生成（1枚あたり数秒〜十数秒）
2. `slides/output/slides/` に `slide-1.png` 〜 `slide-N.png` が出力される

完了したら出力フォルダをユーザーに開いてもらう。

---

## Step 5：確認と修正

生成されたスライドをユーザーに確認してもらう。

修正が必要な場合：

| 修正内容 | 対応 |
|---------|------|
| テキストを変えたい | `slides/image-design.json` を編集 → Step 4 を再実行 |
| 特定のスライドだけ再生成 | `--index` オプションで指定（単体・カンマ・範囲が使える） |
| スタイルを変えたい | brief の「### スタイル」を変更 → Step 3 から再実行 |
| 訴求軸・ターゲットを変えたい | brief の「## 生成パラメータ」を変更 → Step 3 から再実行 |

`--index` の指定例：
```bash
# 3枚目だけ
node slides/engine/generate.js --design slides/image-design.json --out slides/output --index 3

# 5枚目・8枚目・12枚目
node slides/engine/generate.js --design slides/image-design.json --out slides/output --index 5,8,12

# 5枚目・8枚目・10〜12枚目
node slides/engine/generate.js --design slides/image-design.json --out slides/output --index 5,8,10-12
```

再生成が終わったら、講師スライド（10・11番）に顔写真を合成するか、動画素材として使う場合の演出ポイントを Step 6 で案内します。不要なら「以上で完了」で締めくくる。

---

## Step 6：仕上げ（オプション）

### Step 6-A：顔写真の合成

講師の顔写真をスライド上に合成する場合：

```bash
node slides/engine/composite.js \
  --slide slides/output/slides/slide-10.png \
  --photo input/photos/講師名.jpg \
  --out slides/output/slides/slide-10-final.png
```

顔写真は `input/photos/` に置いておく。該当スライド（典型: 10番・11番）ごとに1回ずつ実行する。

### Step 6-B：デモ動画スライドの演出ガイド

スライド8（デモ①：スライド自動生成）・スライド9（デモ②：案件探索自動化）は、あくまで「これからこのデモを見せます」という**導入の静止画**。
動画コンテンツやセミナーで使う際は、このページに到達したら**別画面で実際にツールを動かしている様子に画面を切り替える**演出を推奨。スライド単体で完結させようとせず、ライブデモへの橋渡しとして扱うとインパクトが出る。

ユーザーへの案内例：
```
スライド8・9はデモの導入ページです。動画で使う場合は、ここで画面を切り替えて
ツールが実際に動いている様子を見せてください（スライドはあくまで導入の役割）。
```

## エラーが出たとき

| エラーメッセージ | 対処 |
|----------------|------|
| `ANTHROPIC_API_KEY が設定されていません` | `.env` に `ANTHROPIC_API_KEY=キー` を追加 |
| `GOOGLE_AI_API_KEY が設定されていません` | `.env` に `GOOGLE_AI_API_KEY=キー` を追加 |
| `npm install を実行してください` | `cd slides/engine && npm install` を実行 |
| JSON パースエラー | `slides/image-design-debug.txt` の内容を確認して報告 |

---

## ⚠️ 絶対に守るルール

- `slides/engine/generate.js` と `slides/engine/plan.js` は**絶対に編集しない**
- エラーが出ても、コードを修正しようとしない。エラーメッセージをそのままユーザーに伝えること
- モデル名が「無効」に見えても変更しない。現在のモデル名は正しく設定済み

---

## 注意事項

- 全枚数生成は5〜10分程度かかる
- Gemini API の日本語描画精度は100%ではない（稀に文字が崩れることがある）
- APIの利用料金が発生する（Gemini + Anthropic）
