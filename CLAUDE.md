# cc-slides — AI セールススライド自動生成システム

サービス仕様書（Markdown）を渡すだけで、プロ品質のセールススライド画像を自動生成するツール。

---

## Claudeへの指示

ユーザーがこのプロジェクトを開いたら、以下のステップを順番に能動的に案内する。
ユーザーが何も言わなくても、まずStep 1から始める。

---

## Step 1：セットアップ確認

以下を順番にチェックし、問題があればその場で解決してから次に進む。

### 1-A. node_modules の確認

```bash
ls slides/engine/node_modules
```

存在しなければ実行：
```bash
cd slides/engine && npm install
```

### 1-B. .env の確認

`.env` ファイルが存在するか確認：
```bash
ls -la | grep .env
```

なければ作成を案内：
```bash
cp .env.example .env
```

`.env` を開いて以下の2つのキーが入力済みか確認する：
- `GOOGLE_AI_API_KEY` — 取得先: https://aistudio.google.com/apikey
- `ANTHROPIC_API_KEY` — 取得先: https://console.anthropic.com/

キーが空欄なら、ユーザーにキーの入力を促す。詳細手順は `setup/gemini-api.md` を参照。

セットアップが完了したら「準備完了です。Step 2に進みます」と伝える。

---

## Step 2：スライドの内容を決める

まずユーザーに聞く：

```
どのサービスのスライドを作りますか？

1. Claude Code ブートキャンプ（仕様書あり・すぐ始められます）
2. 別のサービス（一緒に仕様書を作ります）
```

---

### パターンA：Claude Code ブートキャンプの場合

仕様書（`input/brief-template.md`）はすでに用意されている。
いきなり生成せず、まず以下のスライド構成案をユーザーに提示して合意を取る。

**提示するスライド構成案：**

```
以下の構成でスライドを作成しようと思います。
変更・追加・削除があれば教えてください。

01. 表紙（キャッチコピー・セミナー名）
02. 問題提起（給料・物価の現実）
03. 自分ごと化（AI活用5段階）
04. 根本原因（3つの壁）
05. 市場機会（AI人材需給ギャップ）
06. Claude Code とは何か
07. 先行者利益（0.1%の現実）
08. デモ①（スライド自動生成）
09. デモ②（案件探索自動化）
10. 講師①：みかみ 実績
11. 講師②：佐藤将司 実績
12. 1期生の実績
13. 橋渡し（あなたにもできる）
14. 競合との差別化
15. 受講フロー（4ステップ）
16. コンテンツ①：講義13テーマ
17. コンテンツ②：ブートキャンプ
18. コンテンツ③：コミュニティ
19. 本体合計（¥649,000相当）
20. 特典①：社内システム12種
21. 特典②③：1on1・補講
```

ユーザーから「OK」または修正が出たら対応し、合意を確認してから次へ進む。

合意後、スタイルを聞く：

```
デザインスタイルを選んでください。

manus    … 薄ブルー×ネイビー、洗練されたビジネス感（推奨）
stripe   … 純白×ダークネイビー×パープル
apple    … ミニマル、大きな余白
google   … マテリアルカラー、フレンドリー
mckinsey … 白×ネイビー、データ重視
notion   … ソフト、読みやすさ重視
figma    … パープル×ティール、モダン
canva    … パステル、カジュアル
netflix  … ダークテーマ、インパクト重視
nike     … 白黒＋蛍光、エネルギッシュ

```

スタイルが決まったら `input/brief-template.md` の最後の「## スタイル」欄を更新して Step 3 へ進む。

### パターンB：別のサービスの場合

**一問一答でヒアリングして、一緒に仕様書を作る。**
複数の質問を一度に聞かない。1つ答えてもらったら次の質問をする。

1. 「商品名・サービス名を教えてください」
2. 「キャッチコピー（スライド表紙の主役になる1行）を教えてください」
3. 「セミナー名またはサブタイトルはありますか？」
4. 「主催者名を教えてください」
5. 「ターゲット像を教えてください（例：32歳会社員、副業未経験）」
6. 「ターゲットが抱えている悩みを3つ教えてください」
7. 「問題提起に使いたい数字・事実はありますか？（例：実質賃金マイナス傾向など）」
8. 「価格（税抜・税込・分割など）を教えてください」
9. 「価値総額（商品の本来の価値）はいくらですか？」
10. 「ROIを教えてください（例：1日あたりのコスト、何件で回収できるか）」
11. 「講師の情報を教えてください（名前・肩書き・顔写真ファイル名・主な実績）」※複数いれば人数分
12. 「商品の内容（コンテンツ名・説明・価値）を教えてください」
13. 「特典があれば教えてください（名称・価値・内容）」
14. 「デモ動画で見せたい機能はありますか？」
15. 「受講フロー（購入から完了までのステップ）を教えてください」
16. 「競合との差別化ポイントを教えてください」
17. 「受講生・顧客の実績例があれば教えてください」
18. 「デザインスタイルを選んでください（上記の選択肢 or おまかせ）」

全項目が揃ったら `input/brief.md` に保存してから Step 3 へ進む。

## Step 3：スライド生成

### 実行前に必ずユーザーに確認する

コマンドを実行する前に、以下を必ずユーザーに提示して「OK」をもらってから進む。
**確認なしに勝手に実行してはいけない。**

```
以下の内容でスライドを生成します。よろしいですか？

- 仕様書: <パス>
- 商品名: <brief から読み取った商品名>
- スタイル: <brief から読み取ったスタイル>
- 推定枚数: <セールス構成から推定>

「はい」で生成開始します。
```

ユーザーから「はい」または同意を得てから実行する。

### 生成コマンド

仕様書のパスを確認し、以下のコマンドを実行する：

```bash
node slides/engine/run.js --brief <仕様書のパス>
```

例：
```bash
node slides/engine/run.js --brief input/brief.md
```

実行中は以下の流れで進む（ユーザーに説明する）：
1. Claude API がスライド構成とテキストを設計（1〜2分）
2. Gemini API が各スライドを画像生成（1枚あたり数秒〜十数秒）
3. `slides/output/` に `slide-1.png` 〜 `slide-N.png` が出力される

完了したら出力フォルダをユーザーに開いてもらう。

---

## Step 4：確認と修正

生成されたスライドをユーザーに確認してもらう。

修正が必要な場合：

| 修正内容 | 対応 |
|---------|------|
| テキストを変えたい | `slides/image-design.json` を編集 → `--generate-only` で再生成 |
| 特定の1枚だけ再生成 | `--slides` オプションで枚数指定 |
| スタイルを変えたい | 仕様書のスタイル欄を変更 → 再度 `run.js` 実行 |
| 仕様書の内容を変えたい | 仕様書を編集 → 再度 `run.js` 実行 |

再生成コマンド例（設計JSONから画像だけ再生成）：
```bash
node slides/engine/run.js --generate-only
```

---

## Step 5：顔写真の合成（オプション）

講師の顔写真を合成する場合：

```bash
node slides/engine/composite.js \
  --slide slides/output/slide-10.png \
  --photo input/photos/講師名.jpg \
  --out slides/output/slide-10-final.png
```

顔写真は `input/photos/` に置いておく。

## エラーが出たとき

| エラーメッセージ | 対処 |
|----------------|------|
| `ANTHROPIC_API_KEY が設定されていません` | `.env` に `ANTHROPIC_API_KEY=キー` を追加 |
| `GOOGLE_AI_API_KEY が設定されていません` | `.env` に `GOOGLE_AI_API_KEY=キー` を追加 |
| `npm install を実行してください` | `cd slides/engine && npm install` を実行 |
| JSON パースエラー | `slides/image-design-debug.txt` の内容を確認して報告 |

---

## 注意事項

- 全枚数生成は5〜10分程度かかる
- Gemini API の日本語描画精度は100%ではない（稀に文字が崩れることがある）
- APIの利用料金が発生する（Gemini + Anthropic）
